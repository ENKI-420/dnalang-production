## üêõ Known Issues & Limitations

### Current Limitations

1. **LLM Integration:**
   - Gemini API key is hardcoded (should be environment variable)
   - No rate limiting implemented
   - Semantic scoring is simulated (not actual API response)

2. **Data:**
   - All portal data is mock/static
   - No real backend connections
   - IBM Quantum integration is present but not actively querying hardware

3. **Performance:**
   - Three.js scene could be optimized further
   - Large bundle size due to Three.js inclusion
   - No service worker for offline support

4. **Browser Support:**
   - Three.js requires WebGL support
   - Older browsers may not render 3D visualization

### Recommended Fixes

1. Move API keys to environment variables
2. Implement backend API routes in `/app/api/`
3. Add loading states for all async operations
4. Implement error boundaries for robust error handling
5. Add E2E tests with Playwright or Cypress
